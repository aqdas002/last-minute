# M1 Retro — 2026-05-28

**Sprint outcome:** Shipped. 17 commits, 76 backend tests green, frontend builds. End-to-end demo confirmed via curl (backend) + browser HTML check (frontend).

## What worked

- **Spec §0 pivot section** — kept the original product spec intact while documenting the Java/Spring translation. Future readers see the full design history.
- **ArchUnit programmatic style** (plain `@Test` calling `rule.check(classes)`) avoided the `archunit-junit5-engine` SPI flakiness with Spring Boot 3.5 + Surefire 3.5.
- **Per-JVM Testcontainer via static block** (not `@Container`) survived multiple test classes without re-binding ports.
- **`Math.floorDiv((amount * 15) / 100)` boundary tests** — exact-math + parameterized cases caught the 14999999998 → 14999999999 off-by-one in the original spec.
- **Single-source `ReturnToValidator` regex** with 41-case test matrix gave high confidence on a security-sensitive surface.
- **`@JdbcType(PostgreSQLEnumJdbcType.class)`** for named enums — the modern Hibernate 6 pattern; `@JdbcTypeCode(NAMED_ENUM)` casts using the Java class name and Postgres rejects it.
- **JOIN FETCH** in repository @Query methods — DTOs serialize outside the transaction without `LazyInitializationException`.
- **Caffeine cache + `@CacheEvict allEntries`** on admin writes — coarse but correct for M1; cache cleared between integration tests to prevent pollution.

## What didn't (apply to M2)

### Real bugs surfaced by demo (fix in M2)

1. **Bean Validation returns 403 instead of 400** in real HTTP — MockMvc integration tests passed because the filter chain differs. Need a `@RestControllerAdvice` for `MethodArgumentNotValidException` → 400 with structured error body. Add a real HTTP-layer test (not MockMvc) to catch this class of mismatch.
2. **Frontend `<title>` is "frontend"** (Vite default) — change to "Last Minute".
3. **Resend send is fire-and-forget** — no retry on transient failures; no log line indicating the email actually went out (only the dev-mode "would email"). M2 wires real Resend with retry + logged message-id.

### Plan-vs-reality drift (M2 should anticipate)

- **Spring Initializr version drift** — Plans should query Initializr metadata at plan-write time or specify "use latest stable in Maven Central."
- **Initializr `baseDir/` extraction quirk** — pre-`rm -rf` is mandatory if re-extracting.
- **Empty `client-id: ${VAR:}` fails Spring Security validation at boot** — for any OAuth2 registration you reference in YAML, the variable must have a non-empty default OR the registration must be conditional. M2 introduces Google OAuth and must use a `@Configuration` with `@ConditionalOnProperty` rather than YAML alone.
- **JPA enum literals in JPQL** generate Java-class-name casts; pass enums as `@Param`. Already established pattern; carry through M2/M3.
- **Postgres INTERVAL ↔ Java String** needs `@ColumnTransformer(write="?::interval", read="...::text")`. Same pattern applies if M2 adds `admin_actions` with intervals.
- **CHAR vs VARCHAR** mapping needs `@JdbcType(SqlTypes.CHAR)` for Hibernate validation to agree with the schema.
- **CSRF for POST endpoints** — M1 ignored CSRF on `/api/auth`, `/api/webhooks`, `/api/admin`. M2 keeps the same allowlist; M3 introduces CSRF-protected booking endpoints (consumer state-changing).
- **`@Transactional` with thrown exceptions** rolls back by default — `noRollbackFor` needed when an exception path must persist a side effect (e.g., delete-then-throw on expired token).

### Process notes

- **Inline planning + execution** worked better than dispatching subagents for M1 — subagent socket timeouts hit twice (one full plan-write timeout, one execution batch). Inline write is slower from user's POV but more reliable.
- **`sg docker -c '...'` wrapper** is required for `mvn` invocations because the docker group was added to the user after the WSL session started; can drop once the user `newgrp docker` or re-logs in.
- **CI gate not yet exercised against GitHub** — the workflow YAML is committed but unverified; M2 should push a branch + PR to validate it actually runs green.

## Decisions for M2

| Decision | Choice |
|---|---|
| OAuth2 registration | Java `@Configuration` with `@ConditionalOnProperty("auth.google.client-id")` instead of YAML, so empty env vars don't break startup |
| Inngest equivalent | Spring `@TransactionalEventListener` for in-process events; `@Scheduled` for cron; webhook_dead_letter table drained by a `@Scheduled` job. NO external job runner for v1 |
| Bean Validation errors | New `@RestControllerAdvice` with `@ExceptionHandler(MethodArgumentNotValidException.class)` returning 400 + field-level errors |
| Stripe Connect Express webhook handling | `/api/webhooks/stripe` route: verify signature → INSERT into `payment_events` (UNIQUE on `stripe_event_id`) → publish Spring application event. Spring `@TransactionalEventListener` consumes and updates state |
| `admin_actions` audit table | New table introduced in M2 (V3 migration) for the currency-correction tool; reusable in M6 ops surface |
| `payment_events` table | New table in M2 (V3 migration). Originally planned for M3 but the webhook handler arrives in M2 so the table comes with it |
