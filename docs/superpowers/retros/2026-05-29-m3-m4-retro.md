## What went well
- The "fast-path findByX + saveAndFlush" pattern from M2 (avoiding JPA defer-flush hiding DataIntegrityViolationException) generalized cleanly to webhook idempotency.
- AFTER_COMMIT + REQUIRES_NEW eventing pattern paid off: refund + chargeback handlers were one short file each.
- ArchUnit `Clock` rule caught two raw `Instant.now()` slips during M3.

## What hurt
- **JPQL enum literal trap re-appeared.** In `findTodaysConfirmed` I wrote
  `b.status = com.lastminute.bookings.BookingStatus.confirmed` and got
  `'confirmed'::BookingStatus` (Java FQN, not the Postgres enum name).
  Same fix as M2: bind via `@Param` instead. Add a checklist line: **any new
  @Query that compares against an enum MUST use @Param, never a literal.**
- One Edit call hit "File has not been read yet" on a file I'd written
  earlier in the same session. Cost a roundtrip. Pattern: Read-then-Edit
  even for files just Written, when crossing a long pause.
