## What went well

- **Eventing pattern paid dividends.** `@TransactionalEventListener(AFTER_COMMIT)` + `@Transactional(REQUIRES_NEW)` from the M2 retro became the backbone of M9's `RefundRequestAutoCloser` — a brand-new feature wired into an existing webhook with zero changes to the webhook code path.
- **Fast-path + saveAndFlush + DataIntegrityViolation catch** pattern (from the M2 webhook controller) translated directly to M9's refund-request file endpoint. The partial unique index `refund_requests_one_open_per_booking WHERE status = 'open'` makes the "one open at a time" rule a DB constraint instead of a race-prone application check.
- **Status pills** (M7 + M11) are the same React component shape — 5 status × style map. Cheap and consistent UX language across the app.
- **The retro lesson stuck.** Every new `@Query` in M5–M14 binds enum values via `@Param`. Zero new instances of the `'confirmed'::BookingStatus` trap. Lessons get written down for a reason.

## What hurt

- **JPQL enum literal trap, third time.** Caught in M5's `findRemindersDue` AND `findExpiredPending`. Both were pre-existing M2 queries that happened never to be exercised against a real query path until M5 added the reminder job test. Lesson: **integration test every repository method, not just the entity**.
- **CI rescue chain (4 commits).** Going public exposed problems that local dev hid:
  1. `mvnw` mode `100644` in git — WSL doesn't preserve exec bit. Fix: `git update-index --chmod=+x`.
  2. ESLint 10.4 needs Node 22 (`util.styleText`). CI ran Node 20. Fix: bump.
  3. Vite 8's `rolldown` native binding wasn't in the lockfile (generated on WSL). Tried two workarounds (lockfile cross-arch, `node-linker=hoisted`); neither solved the relative-require inside rolldown's `binding.mjs`.
  4. **Real fix:** downgrade Vite 8 → 7 (rollup classic = pure JS, no native binding). Plug-in dropped 6 → 5.
- **One Edit failure mid-session** ("File has not been read yet" on a file I'd Written earlier). Pattern: **Read-then-Edit even for self-written files** when crossing a long pause.
- **React 19 purity rule** blocked `Date.now()` in render (M13's no-show button). Fix: use TanStack Query's `dataUpdatedAt` as a pure "now" reference.
- **`setState`-in-effect rule** blocked the previous `useEffect`-to-default-category pattern (M11 lint pass). Fix: derive from list directly, no effect.
